# Prompt测试优化方案

## 方案A：批量推理（Batch Inference）- 已实现 ✅

### 原理
GPU擅长并行计算。处理1句话和处理50句话的耗时在1.5B模型上差异极小。

### 实现
- 新增 `batch_predict()` 函数：一次性处理所有文本
- 修改 `run_benchmark()`：使用批量推理替代串行推理
- 自动padding和批处理

### 性能提升
- **预期**：测试时间从6秒缩短到0.5秒以内
- **实际效果**：取决于数据量和GPU性能

### 使用方法
```python
# 默认使用批量推理
run_benchmark(use_batch=True)

# 如需对比，可以使用串行推理
run_benchmark(use_batch=False)
```

### 技术细节
- 使用 `tokenizer(prompts, padding=True)` 自动padding
- 一次性生成所有结果
- 分别解析每个输出

## 方案B：DSPy自动优化（Automated Optimization）- 可选实现

### 原理
DSPy是斯坦福大学开发的"Prompt编程"框架，可以：
- 自动尝试数千种Prompt组合
- 像训练神经网络一样"训练"Prompt
- 找到准确率最高的"神级Prompt"

### 预期效果
- 准确率从70%提升到90%+
- 自动找到最佳Prompt格式

### 安装
```bash
pip install dspy-ai
```

### 使用方法
```python
# 运行DSPy优化
python spider/dspy_optimization.py
```

### 工作流程
1. 定义输入输出格式
2. 提供训练数据（前30条）
3. DSPy自动尝试数千种Prompt组合
4. 在验证集上测试（剩余数据）
5. 返回准确率最高的Prompt

### 注意事项
- 优化过程可能需要几分钟到几十分钟
- 需要足够的训练数据
- 适合长期优化，不适合快速迭代

---

## 推荐使用策略

### 快速测试和迭代
**使用方案A（批量推理）**
- 快速验证多个Prompt的效果
- 快速对比不同配置
- 适合开发阶段

### 长期优化
**使用方案B（DSPy）**
- 当有足够数据时
- 需要找到最佳Prompt时
- 适合生产环境优化

### 组合使用
1. 先用方案A快速测试多个Prompt候选
2. 选出表现较好的几个Prompt
3. 用方案B对这些Prompt进行深度优化
