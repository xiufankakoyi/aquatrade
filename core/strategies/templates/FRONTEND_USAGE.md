# 前端使用指南 - AI 策略生成器

## 访问方式

### 方式1: 直接访问路由
在浏览器中访问：
```
http://localhost:5173/strategy-generator
```
（假设前端开发服务器运行在 5173 端口）

### 方式2: 通过导航菜单
在侧边栏或导航菜单中添加链接（需要手动添加）

## 使用步骤

### 1. 填写策略信息

**策略名称**（可选）：
- 默认值：`AI策略`
- 示例：`AI双均线策略`、`突破策略`、`RSI超买超卖策略`

**策略描述**（必填）：
- 用自然语言描述你的策略逻辑
- 要清晰具体，包含买入和卖出条件

### 2. 示例描述

#### 示例1: 双均线策略
```
写一个策略，当5日均线上穿20日均线时买入，下穿时卖出。
```

#### 示例2: 突破策略
```
股价突破20日均线买入，RSI大于70卖出，最多持仓5天。
```

#### 示例3: RSI 超买超卖
```
RSI小于30买入，大于70卖出，亏损超过10%止损。
```

#### 示例4: MACD 策略
```
MACD金叉买入，死叉卖出，持仓不超过10天。
```

### 3. 生成策略

1. 点击 **"🤖 生成策略"** 按钮
2. 等待生成完成（通常需要几秒到几十秒）
3. 生成成功后，会显示文件名和保存路径

### 4. 使用生成的策略

生成成功后：
1. 策略文件已保存到 `core/strategies/user/` 目录
2. 文件名格式：`ai_gen_<timestamp>.py`
3. 可以在策略列表中选择该策略进行回测

## API 调用方式

### 前端代码示例

```typescript
import { apiService } from '@/services/api';

// 生成策略
try {
  const result = await apiService.generateStrategy(
    '写一个策略：股价突破20日均线买入，RSI大于70卖出',
    'AI突破策略'
  );
  
  console.log('生成成功:', result.filename);
  console.log('消息:', result.message);
} catch (error) {
  console.error('生成失败:', error);
}
```

### API 接口

**端点**: `POST /api/strategies/generate`

**请求体**:
```json
{
  "description": "写一个策略：股价突破20日均线买入，RSI大于70卖出",
  "name": "AI突破策略"
}
```

**响应**:
```json
{
  "success": true,
  "data": {
    "filename": "ai_gen_1712345678.py",
    "message": "策略已成功生成并保存为 ai_gen_1712345678.py"
  }
}
```

**错误响应**:
```json
{
  "success": false,
  "error": "策略描述不能为空"
}
```

## 注意事项

### 1. LLM 服务配置

确保 LLM 服务正在运行：
- 默认地址：`http://127.0.0.1:1234/v1`
- 可以在 `config/config.py` 中修改配置

### 2. 生成时间

- 生成时间取决于 LLM 服务的响应速度
- 通常需要 5-30 秒
- 如果超时，请检查 LLM 服务是否正常运行

### 3. 代码质量

- 生成的代码需要经过验证
- 建议在回测前检查生成的策略代码
- 可以手动编辑生成的策略文件进行优化

### 4. 策略描述建议

**好的描述**：
- ✅ "当5日均线上穿20日均线时买入，下穿时卖出"
- ✅ "股价突破20日均线买入，RSI大于70卖出，最多持仓5天"
- ✅ "RSI小于30买入，大于70卖出，亏损超过10%止损"

**不好的描述**：
- ❌ "写一个赚钱的策略"（太模糊）
- ❌ "买入股票"（缺少卖出条件）
- ❌ "用AI分析"（不够具体）

## 故障排查

### 问题1: 生成失败，提示 "LLM 调用失败"

**解决方案**：
1. 检查 LLM 服务是否运行
2. 检查 `config/config.py` 中的 `LLM_API_BASE` 配置
3. 查看后端日志获取详细错误信息

### 问题2: 生成超时

**解决方案**：
1. 检查网络连接
2. 增加超时时间（需要修改代码）
3. 使用更快的 LLM 模型

### 问题3: 生成的代码有错误

**解决方案**：
1. 检查策略描述是否清晰
2. 查看生成的代码文件
3. 手动修复代码错误
4. 重新生成或手动编写策略

## 生成的策略文件位置

所有生成的策略文件保存在：
```
core/strategies/user/ai_gen_<timestamp>.py
```

文件会自动被策略工厂识别，可以在策略列表中选择使用。
