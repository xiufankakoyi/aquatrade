## 修复前端显示中文乱码问题

### 问题原因

* subprocess 读取子进程输出时编码不匹配

* 子进程 Python 脚本在 Windows 上默认使用 GBK 编码

### 修复方案

**1. 修改 main\_launcher.py** (添加 UTF-8 编码设置)

```python
import sys
import io
sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8', errors='replace')
sys.stderr = io.TextIOWrapper(sys.stderr.buffer, encoding='utf-8', errors='replace')
```

**2. 修改 server.py** (Popen 添加环境变量)

```python
env = os.environ.copy()
env['PYTHONIOENCODING'] = 'utf-8'
process = subprocess.Popen(
    cmd,
    shell=True,
    stdout=subprocess.PIPE,
    stderr=subprocess.STDOUT,
    bufsize=1,
    text=True,
    encoding='utf-8',
    errors='replace',
    cwd=BASE_DIR,
    creationflags=subprocess.CREATE_NO_WINDOW,
    env=env  # 添加环境变量
)
```

**3. 修改 run\_crawler.py** (添加 UTF-8 编码)

```python
import os
os.environ['PYTHONIOENCODING'] = 'utf-8'
```

### 测试步骤

1. 重启 server.py
2. 提交爬取任务
3. 检查前端 output 是否正确显示中文
