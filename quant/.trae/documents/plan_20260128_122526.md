# 自动 Debug 计划：前端爬取进度显示

## 问题分析
当前系统存在以下问题：
1. 前端提交请求后，爬虫在后端运行但前端无法看到详细进度
2. 状态轮询可能存在问题，无法正确显示爬取过程

## 修复步骤

### 1. 检查并修复前端状态轮询逻辑
- 检查 `/api/status` 接口是否返回进度信息
- 增强前端状态显示，显示爬取的 API 列表和进度

### 2. 增强后端状态返回
- 在任务状态中包含当前正在爬取的 API 名称
- 返回已完成的 API 列表和进度百分比

### 3. 实时进度更新
- 后端在爬取每个 API 时输出进度
- 前端每秒轮询并更新显示

### 4. 测试完整流程
- 启动服务器
- 通过前端提交爬取请求
- 验证前端显示爬取进度

## 修改文件
- `server.py` - 增强状态返回
- `web/app.js` - 改进状态显示
