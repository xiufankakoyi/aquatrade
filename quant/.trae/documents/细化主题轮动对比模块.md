## 需要实现的三大功能

### 功能1：细化主题轮动对比
**目标格式**：
```
【主题轮动】：
  主线：商业航天（第3日，29只）, AI应用（第2日，10只）
  支线：AI营销（第2日，8只）
  新晋：固态电池（首日，6只）
  退潮：消费（从12只降至0只）
```

**实现方式**：
- 从 market_sentiment_cycle.json 的 data 列表追踪主题历史
- 计算每个主题的持续天数
- 主线取 count 前2-3且持续2日以上的主题

### 功能2：添加梯队名单
**目标格式**：
```
【梯队名单】：
  4板：韩建河山（公告）、xxx（题材）
  3板：弘讯科技（可控核聚变）、xxx
  2板：中国卫通（商业航天）、xxx、xxx
  1板：共47只（详见核心板块）
```

**实现方式**：
- 从 ladder_detail['dates'][0]['boards'] 获取
- 按 level 分组显示股票名称和题材

### 功能3：恢复核心板块个股详情（原 Section 8.2）
**目标格式**：
```
【核心板块与个股详情】：

主线一：固态电池（6只）
  杉杉股份(600884) 1板：业绩预增+负极龙头
    封单1.2亿/封成15%/封流8%
    实流3.5亿 题材:锂电池

主线二：公告/重组（4只）
  韩建河山(603616) 4板：控制权变更
    ...
```

**实现方式**：
- 从 sector_heat 取前3主线题材
- 从 limit_up_filter 获取该题材下的所有涨停股
- 显示每只股票的：名称、代码、连板数、逻辑、封单数据
