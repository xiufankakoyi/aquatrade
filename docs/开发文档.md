## 系统概述

AquaTrade 是一个高性能、事件驱动的量化交易与回测平台，采用现代化的技术栈，支持 CPU 和 GPU 高度并行的计算优化。

### 核心特性

- **性能至上**: 深度集成 QuestDB、Polars、NumPy 和 Numba，针对量化回测的计算密集型场景优化
- **流式架构**: 回测引擎采用生成器模式，实时推送结果，支持超大规模数据集的低延迟反馈
- **组件化与解耦**: 数据查询、回测执行、策略定义、可视化呈现均高度模块化
- **三层策略系统**: 从配置化到完全自定义，满足不同开发需求
- **声明式因子系统**: 策略只需声明所需因子，系统自动注入数据

### 技术栈

| 层级 | 技术选型 |
|------|---------|
| 前端 | Vue 3 + Vite + ECharts + Socket.IO |
| 后端 | Python + FastAPI/Flask + Granian |
| 数据库 | QuestDB (热数据) + Parquet (冷数据) + ArcticDB |
| 计算 | NumPy + Polars + Numba (JIT) |
| 存储 | Parquet + Arrow (零拷贝) |

## 回测系统

### 统一回测引擎 (UnifiedBacktestEngine)

回测引擎是系统的核心，支持向量化计算、配置化驱动和多粒度回测。

#### 核心特性

1. **向量化优先**: 所有计算尽可能使用 NumPy/Pandas/Polars 向量化
2. **配置化驱动**: 策略可以是 Python 类或 YAML/JSON 配置
3. **多粒度支持**: 日线、分钟线、Tick级别
4. **统一接口**: 无论策略形式如何，都使用相同的 API
5. **流式输出**: 支持实时推送回测进度

#### 使用示例

```python
from core.backtest.unified_engine import UnifiedBacktestEngine, BacktestConfig
from core.strategies.strategy_layers import DualMAStrategy

# 创建引擎
config = BacktestConfig(
    initial_capital=1_000_000.0,
    commission_rate=0.0003,
    position_ratio=0.1
)
engine = UnifiedBacktestEngine(data_query, config)

# 创建策略
strategy = DualMAStrategy(fast=5, slow=10)

# 运行回测（流式）
for event in engine.run_backtest('2020-01-01', '2024-01-01', strategy):
    if event['type'] == 'backtest_progress':
        print(f"进度: {event['data']['progress']}%")
    elif event['type'] == 'trade':
        print(f"交易: {event['data']}")
    elif event['type'] == 'backtest_complete':
        result = event['data']
        print(f"总收益: {result['total_return']}%")
```

### 因子矩阵 (FactorMatrix)

因子矩阵是回测引擎的核心数据结构，用于高效存储和访问因子数据。

#### 数据结构

```python
@dataclass
class FactorMatrix:
    """
    因子矩阵数据结构
    
    Attributes:
        values: 三维数组 (T, N, F) - 日期 × 股票 × 因子
        dates: 日期列表 (长度 T)
        codes_int: 股票代码 int32 数组 (长度 N)
        codes_str: 股票代码字符串列表 (长度 N)
        factor_names: 因子名列表 (长度 F)
        date_to_idx: 日期到索引的映射
        code_to_idx: 股票代码到索引的映射
    """
    values: np.ndarray
    dates: List[str]
    codes_int: np.ndarray
    codes_str: List[str]
    factor_names: List[str]
    date_to_idx: Dict[str, int]
    code_to_idx: Dict[str, int]
```

#### 性能优化

1. **Arrow 零拷贝转换**: 使用 Polars + Arrow 避免数据拷贝
2. **股票代码 int32 存储**: 减少 50% 内存占用
3. **Parquet 缓存**: 基于参数哈希的智能缓存
4. **内存映射**: 大数据集使用 np.memmap 避免内存溢出

### 三层策略系统

系统提供三个层级的策略开发方式，从简单到高级：

```
┌─────────────────────────────────────────────────────────────┐
│                    策略开发难度层级                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  第一层：配置策略（最简单）                                    │
│  └── YAML 配置文件，无需写代码                                │
│                                                             │
│  第二层：函数策略（中等）                                      │
│  └── 简单 Python 函数，逐日调用                               │
│                                                             │
│  第三层：类策略（高级）                                        │
│  ├── SimpleStrategy：声明式 + 内置信号函数                    │
│  └── VectorizedStrategyBase：完全控制                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 第一层：配置策略（YAML）

适用于：非程序员、快速验证策略思路

配置文件示例（`configs/my_strategy.yaml`）：

```yaml
strategy:
  name: "双均线策略"
  
  parameters:
    - name: fast_window
      type: int
      default: 5
    - name: slow_window
      type: int
      default: 20

  indicators:
    - name: ma_fast
      type: ma
      params:
        column: close
        window: "${fast_window}"
    - name: ma_slow
      type: ma
      params:
        column: close
        window: "${slow_window}"

  rules:
    - action: buy
      condition: "crossover(ma_fast, ma_slow)"
    - action: sell
      condition: "crossunder(ma_fast, ma_slow)"
```

### 第二层：函数策略

适用于：需要状态管理、逻辑较简单的策略

```python
from core.strategies.strategy_layers import FunctionStrategy, Position

def my_strategy(date: str, factors: dict, position: Position, history: list):
    """
    策略函数
    
    Args:
        date: 当前日期 (YYYY-MM-DD)
        factors: 当日因子值字典
        position: 当前持仓状态
        history: 历史交易记录
    
    Returns:
        tuple: (action, ratio)
            - action: 'buy', 'sell', 'hold'
            - ratio: 买入仓位比例 (0-1)，卖出时为 None
    """
    # 买入条件
    if not position.has_position:
        if factors['ma5'] > factors['ma10'] and factors['volume'] > 1e6:
            return 'buy', 0.1  # 买入 10% 仓位
    
    # 卖出条件
    else:
        if factors['ma5'] < factors['ma10']:
            return 'sell', None  # 全卖
    
    return 'hold', None

# 创建策略实例
strategy = FunctionStrategy(
    my_strategy,
    required_factors=['ma5', 'ma10', 'volume']
)
```

### 第三层：类策略（推荐）

#### SimpleStrategy（简化版）

适用于：大多数策略，代码简洁

```python
from core.strategies.strategy_layers import SimpleStrategy
from core.strategies.utils.signal_utils import crossover, crossunder

class MyStrategy(SimpleStrategy):
    """我的策略"""
    
    # 声明所需因子
    required_factors = ['ma5', 'ma10', 'rsi_14']
    
    def _generate_signals(self, factors, trading_dates, stock_codes):
        """
        生成信号矩阵
        
        Args:
            factors: 因子字典 {name: matrix(T, N)}
            trading_dates: 日期列表
            stock_codes: 股票代码列表
        
        Returns:
            signals: 信号矩阵 (T, N), 0=hold, 1=buy, 2=sell
        """
        T, N = len(trading_dates), len(stock_codes)
        signals = np.zeros((T, N), dtype=np.int32)
        
        ma5 = factors['ma5']
        ma10 = factors['ma10']
        rsi = factors['rsi_14']
        
        # 金叉 + RSI 超卖 -> 买入
        golden = crossover(ma5, ma10)
        oversold = rsi[1:] < 30
        signals[1:][golden & oversold] = 1
        
        # 死叉 -> 卖出
        death = crossunder(ma5, ma10)
        signals[1:][death] = 2
        
        return signals
```

```python
        # 生成信号

        signals = np.zeros((T, N), dtype=np.int32)
        
        # ... 策略逻辑
        
        return signals
```

## 开发规范

### 代码原则

1. **DRY 原则**: 避免重复代码，通过抽象公共逻辑提升可维护性
2. **KISS 原则**: 保持简单，拒绝过度设计，优先选择直观的解决方案
3. **防御性编程**: 覆盖边缘情况，对输入参数进行校验，对异常进行合理处理

### 命名规范

- 变量、函数、类名统一使用英文命名
- 遵循 PEP8 编码规范
- 命名应具有自描述性，避免无意义的缩写

### 注释规范

- 所有函数和类必须包含 DocString，说明功能、参数与返回值
- 复杂逻辑需要添加行内注释
- 禁止注释代码（用 Git 版本控制）

### 测试规范

- 测试脚本仅限 `sandbox/` 目录
- 严禁 `server/` 内测试文件
- 开发流程：搜索现有实现 → sandbox验证 → 自动化测试 → 代码审查 → 提交代码

### 沟通规范

- 语言：中文简体
- 需求不明主动澄清
- 发现 Bug/漏洞立即指出

### 代码提交

- 确保每次提交通过所有测试
- 提交信息清晰描述变更内容与原因
- 遵循小步提交原则

## 性能优化

### Numba JIT 加速

核心计算函数使用 Numba JIT 加速：

```python
from numba import jit, njit

@njit(cache=True, fastmath=True)
def calculate_drawdown(equity_values: np.ndarray) -> float:
    """Numba 加速的最大回撤计算"""
    n = len(equity_values)
    if n == 0:
        return 0.0
    
    peak = equity_values[0]
    max_dd = 0.0
    
    for i in range(n):
        if equity_values[i] > peak:
            peak = equity_values[i]
        if peak > 0:
            dd = (equity_values[i] - peak) / peak * 100
            if dd < max_dd:
                max_dd = dd
    
    return max_dd
```

### Polars 向量化处理

使用 Polars 进行大规模数据处理：

```python
import polars as pl

# 读取 Parquet
df = pl.scan_parquet("data/parquet_data/factors_momentum.parquet")

# 向量化操作
result = (
    df
    .filter(pl.col("trade_date") >= "2020-01-01")
    .group_by("stock_code")
    .agg([
        pl.col("rsi_14").mean().alias("avg_rsi"),
        pl.col("volume").sum().alias("total_volume")
    ])
    .collect()
)
```

## 常见问题

### Q: 如何添加新的因子？

A: 在 `required_factors` 列表中添加因子名称即可：

```python
class MyStrategy(SimpleStrategy):
    required_factors = ['ma5', 'ma10', 'new_factor']  # 添加新因子
```

如果因子不在预计算列表中，需要在 `FactorCalculator` 中实现计算逻辑。

### Q: 如何处理 NaN 值？

A: 使用 NumPy 的 NaN 处理函数：

```python
# 检查 NaN
valid = ~np.isnan(close)

# 忽略 NaN 计算
mean_val = np.nanmean(close, axis=0)

# 填充 NaN
filled = np.nan_to_num(close, nan=0.0)
```

### Q: 如何实现止损/止盈？

A: 在策略中添加条件判断：

```python
def _generate_signals(self, factors, trading_dates, stock_codes):
    # ...
    
    # 止损：跌幅超过 8%
    stop_loss = (close / cost_price - 1) < -0.08
    
    # 止盈：涨幅超过 20%
    take_profit = (close / cost_price - 1) > 0.20
    
    sell_condition = stop_loss | take_profit
    signals[sell_condition] = 2
```

### Q: 函数策略和类策略如何选择？

A: 
- **函数策略**: 适合简单逻辑、需要状态管理（如持仓天数）
- **类策略**: 适合复杂逻辑、需要向量化计算、性能要求高
